# 🚀 Flask + Apache2 웹 서버 배포 실전 가이드

이 가이드 문서는 팀원들이 상황에 맞게 안정적인 웹 서버 환경을 구축할 수 있도록 두 가지 배포 방식을 안내합니다. 도커를 활용한 컨테이너 기반 배포 방식과, 관리자(root) 권한 없이 리눅스 로컬 계정만으로 배포하는 방식을 모두 다룹니다.

---

## 🐳 방법 1: Docker를 활용한 배포 (Flask + Gunicorn + Apache2)

이 방식은 도커 환경에서 `Gunicorn`을 WSGI로 사용하고, `Apache2`를 리버스 프록시로 설정하는 안정적인 아키텍처입니다.

### 1-1. 프로젝트 폴더 구조 만들기
작업을 시작하기 전에 프로젝트 폴더를 만들고, 아래와 같은 구조로 빈 파일들을 생성해 주세요.

```plaintext
my-project/
├── app/
│   ├── app.py               # Flask 애플리케이션 코드
│   └── requirements.txt     # 설치할 파이썬 라이브러리 목록
├── Dockerfile               # Flask+Gunicorn 컨테이너용 레시피
├── my-site.conf             # Apache2 리버스 프록시 설정
└── docker-compose.yml       # 전체 컨테이너 오케스트레이션 설정
```

### 1-2. 주방 세팅 (Flask 앱 및 의존성 작성)
먼저 요리가 될 핵심 코드와 재료 명세서를 작성합니다.

**`app/requirements.txt`**
이곳에 우리의 통역사 역할을 해줄 `gunicorn`을 반드시 추가해야 합니다.
```plaintext
Flask==3.0.0
gunicorn==21.2.0
```

**`app/app.py`**
간단하게 화면에 인사를 띄우는 코드를 작성합니다.
```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return "안녕하세요! 도커 환경에서 Apache2와 Gunicorn을 거쳐 실행된 Flask 앱입니다."

if __name__ == '__main__':
    # Gunicorn을 사용할 것이므로 아래 내장 서버 실행 코드는 무시됩니다.
    app.run(host='0.0.0.0', port=5000)
```

### 1-3. 밀키트 레시피 작성 (Dockerfile)
우리의 파이썬 코드를 실행해 줄 환경을 정의합니다. 앞서 설명한 대로 테스트용 주방보조가 아닌, 전문 셰프(Gunicorn)를 실행하도록 지시합니다.

**`Dockerfile`**
```dockerfile
# 1. 파이썬 3.9 환경 가져오기
FROM python:3.9-slim

# 2. 작업할 폴더 지정
WORKDIR /app

# 3. 우리 앱 코드를 도커 안으로 복사
COPY ./app /app

# 4. 필요한 라이브러리 설치 (Flask, gunicorn)
RUN pip install --no-cache-dir -r requirements.txt

# 5. Gunicorn(WSGI) 서버 실행 (포트 5000, 작업자 4명 배치)
CMD ["gunicorn", "--workers", "4", "--bind", "0.0.0.0:5000", "app:app"]
```

### 1-4. 홀 매니저 지시서 작성 (my-site.conf)
Apache2 서버가 외부 트래픽을 받아 도커 내부의 Flask 컨테이너로 넘겨주도록(리버스 프록시) 설정합니다.

**`my-site.conf`**
```apache
<VirtualHost *:80>
    ServerName localhost

    # 도커 내부 네트워크에서 'flask-app'이라는 이름의 컨테이너 5000번 포트로 모든 요청을 전달
    ProxyPreserveHost On
    ProxyPass / http://flask-app:5000/
    ProxyPassReverse / http://flask-app:5000/

    ErrorLog /usr/local/apache2/logs/error.log
    CustomLog /usr/local/apache2/logs/access.log common
</VirtualHost>
```
> [!WARNING]
> 주의: 실제 도커 공식 `httpd` 이미지를 사용할 때는 `mod_proxy` 및 `mod_proxy_http` 모듈이 활성화된 `httpd.conf` 메인 설정 파일도 함께 준비해 주어야 합니다. 이 가이드에서는 프록시 설정의 핵심 로직만 다룹니다.

### 1-5. 이삿짐 센터 소장님 호출 (docker-compose.yml)
마지막으로 주방(Flask)과 홀(Apache2)을 묶어서 한 번에 실행하게 해주는 컴포즈 파일을 작성합니다.

**`docker-compose.yml`**
```yaml
version: '3.8'

services:
  # 1. 주방 (백엔드 앱)
  flask-app:
    build: .                 # 현재 폴더의 Dockerfile을 사용해 빌드
    expose:
      - "5000"               # 외부에 공개하지 않고 내부 네트워크(Apache)에만 5000번 포트 개방

  # 2. 홀 매니저 (웹 서버)
  apache-server:
    image: httpd:2.4         # 공식 아파치 이미지 사용
    ports:
      - "80:80"              # 사용자의 브라우저(80)와 아파치 컨테이너(80) 연결
    volumes:
      - ./my-site.conf:/usr/local/apache2/conf/extra/my-site.conf
    depends_on:
      - flask-app            # flask-app이 먼저 켜진 후 아파치가 켜지도록 순서 보장
```

### 1-6. 실행 및 테스트 (Action!)
모든 파일 준비가 끝났습니다. 이제 터미널을 열고 `my-project` 폴더로 이동한 뒤, 아래의 마법 같은 명령어 한 줄을 입력하세요.

```bash
docker-compose up -d --build
```
- **`up`**: 컨테이너들을 실행해라.
- **`-d`**: 백그라운드에서 조용히 실행해라 (터미널을 꺼도 계속 돌아감).
- **`--build`**: 코드가 바뀌었을 수 있으니 이미지를 새로 구워서(빌드해서) 실행해라.

성공적으로 실행되었다면 웹 브라우저를 열고 `http://localhost` 에 접속해 보세요. 우리가 `app.py`에 작성했던 인사말이 화면에 예쁘게 출력하면서 완료됩니다!

## 👤 방법 2: 관리자 권한 없이 로컬 계정으로 배포 (Apache2 mod_wsgi)

이 방식은 서버 내 도커를 사용할 수 없는 경우, `root` 권한 없이 내 계정(홈 디렉토리)만으로 Apache2와 Flask를 연동하여 서빙하는 방법입니다.

> [!IMPORTANT]
> - **목표**: 시스템 설정 파일(`/etc/apache2`)을 건드리지 않고, 내 폴더에 설정 파일을 만들어 WSGI 모듈로 Apache와 Flask를 연결합니다.
> - **주의**: 모든 코드의 `[본인계정]`과 `[포트번호]`는 반드시 팀에 할당된 정보로 변경해야 합니다! 포트는 **1024번 이상(Unprivileged Port)**을 사용합니다.

### 2-1. 작업 디렉토리 생성 및 가상환경 준비
서버 구동에 필요한 폴더를 만들고 Python 가상환경을 세팅합니다.

```bash
# 1. 서버 전용 폴더 및 로그/실행 폴더 생성
mkdir -p ~/my-apache/app
mkdir -p ~/my-apache/logs
mkdir -p ~/my-apache/run

# 2. 작업 폴더로 이동
cd ~/my-apache

# 3. 파이썬 가상환경(venv) 생성 및 활성화
python3 -m venv venv
source venv/bin/activate

# 4. Flask 설치
pip install flask
```

### 2-2. Flask 애플리케이션 작성 (`app.py`)
웹에서 보여줄 간단한 파이썬 코드를 작성합니다.

```bash
# 편집기로 app.py 열기
nano ~/my-apache/app/app.py
```

```python
# --- app.py 내용 ---
from flask import Flask
app = Flask(__name__)

@app.route("/")
def hello():
    return "<h1>Hello Flask! (Team: [본인계정])</h1>"

if __name__ == "__main__":
    app.run()
```

### 2-3. WSGI 연결 파일 작성 (`app.wsgi`)
Apache가 Flask 앱을 인식하고 실행할 수 있도록 다리(Bridge) 역할을 하는 파일을 만듭니다.

```bash
# 편집기로 app.wsgi 열기
nano ~/my-apache/app/app.wsgi
```

```python
# --- app.wsgi 내용 ---
import sys
import logging

logging.basicConfig(stream=sys.stderr)

# Flask 앱이 있는 경로 추가 (본인 계정명으로 변경!)
sys.path.insert(0, '/home/[본인계정]/my-apache/app')

# app.py에서 만든 app 객체를 application으로 가져오기
from app import app as application
```

### 2-4. Apache 설정 파일 작성 (`apache2.conf`)
나만의 아파치 설정을 만듭니다. 시스템 설정을 건드리지 않고 이곳에서 모든 서버 설정을 처리합니다.

```bash
nano ~/my-apache/apache2.conf
```

```apache
# --- apache2.conf 내용 (계정명, 포트번호 수정 필수!) ---

# 1. 포트 설정 (팀별로 부여받은 1024 이상의 포트 입력)
Listen [포트번호]

# 2. 경로 설정
ServerRoot "/home/[본인계정]/my-apache"
PidFile "/home/[본인계정]/my-apache/run/apache2.pid"
ErrorLog "/home/[본인계정]/my-apache/logs/error.log"

# 3. 필수 모듈 로드
LoadModule mpm_event_module /usr/lib/apache2/modules/mod_mpm_event.so
LoadModule authz_core_module /usr/lib/apache2/modules/mod_authz_core.so
LoadModule wsgi_module /usr/lib/apache2/modules/mod_wsgi.so

# 4. WSGI 및 Flask 연동 설정
WSGIDaemonProcess team_app threads=5 python-home=/home/[본인계정]/my-apache/venv
WSGIScriptAlias / /home/[본인계정]/my-apache/app/app.wsgi

<Directory "/home/[본인계정]/my-apache/app">
    WSGIProcessGroup team_app
    WSGIApplicationGroup %{GLOBAL}
    Require all granted
</Directory>
```

### 2-5. 서버 실행 및 접속 테스트
모든 설정이 완료되었습니다. 아파치 데몬을 직접 실행해 봅시다.

**서버 켜기 (Start)**
```bash
/usr/sbin/apache2 -d ~/my-apache -f ~/my-apache/apache2.conf -k start
```

**서버 끄기 (Stop)**
```bash
/usr/sbin/apache2 -d ~/my-apache -f ~/my-apache/apache2.conf -k stop
```

**접속 확인**
웹 브라우저를 열고 `http://[서버IP]:[조포트번호]` 로 접속하여 페이지가 뜨는지 확인합니다.
> [!CAUTION]
> 에러가 발생했다면 터미널에서 `cat ~/my-apache/logs/error.log` 를 입력해 바로 원인을 확인하세요!
