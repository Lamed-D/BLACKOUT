# 프로젝트 개발계획서 (Project Development Plan)

## 1. 프로젝트 개요
**1.1 프로젝트 명**
- 미정 (웹 서비스 구축 프로젝트)

**1.2 개발 목적**
- 웹서비스의 전체 아키텍처(프론트엔드-백엔드-DB) 구축 및 실무 수준의 시스템 연동 역량 강화
- 회원, 게시판, 관리자 기능을 포함한 핵심 비즈니스 로직 설계 및 구현
- 의도적 취약점 환경 구성을 통한 웹 보안 및 취약점 분석 실습 플랫폼 확보

**1.3 기대 효과**
- 로컬 개발 환경을 넘어선 실서버 수준의 유저 권한 기반 인프라 배포 경험 확보
- 시스템 로그 분석 및 트러블슈팅을 통한 백엔드 서버 운영 능력 향상
- 프론트엔드-백엔드 분업화 및 코드 리뷰 기반의 체계적인 협업 프로세스 확립

## 2. 개발 환경 및 기술 스택
**2.1 기술 스택**
- **Frontend:** HTML/HTML5, CSS, JavaScript (바닐라 JS 중심)
- **Backend/Web Server:** Python (Flask) + **Apache2 (mod_wsgi)**
- **Database:** **MariaDB (또는 MySQL)**
- **Infrastructure:** **Docker**, **Docker Compose**

**2.2 시스템 요구사항 및 제약사항**
- **포트 설정:** 최고 관리자(root) 권한 없이 일반 계정만으로 웹 서버를 서빙할 수 있도록 **1000번대 이상(1024 이상)의 Unprivileged Port**를 할당하여 사용.
- **도커 환경:** `docker-compose.yml`을 활용하여 시스템 인프라를 도커라이징(Dockerizing)하여 배포 환경의 독립성과 이식성 확보.
- **DB 분리:** MariaDB를 도커 환경 내에서 별도의 데이터베이스 전용 컨테이너로 분리 구동하고, 백엔드 컨테이너/서버와 네트워크 연동.
- **웹 서버 연동:** Flask 애플리케이션의 단독 실행(`app.run`)을 지양하고, 웹 서버 엔진으로 Apache2를 도입하여 `mod_wsgi`를 통한 브릿지 연동 구성.

**2.3 유저 권한 웹 서버 배포 전략 (Apache2 + Flask)**
- **사용자 독립 환경 구성:** 시스템 기본 설정 디렉토리(`/etc/apache2`)를 수정하지 않고, 사용자 홈 디렉토리(예: `~/my-apache`) 기준의 커스텀 웹 서버 디렉토리 구성.
- **커스텀 설정 파일 구축:** `ServerRoot`, `PidFile`, `ErrorLog` 등 아파치 데몬 실행을 위한 모든 경로를 서빙 사용자 권한 디렉토리로 지정한 커스텀 `apache2.conf` 세팅.
- **가상환경 연동:** 파이썬 가상환경(`venv`)과 `mod_wsgi`를 활용하여 Flask 진입점(`app.wsgi`)을 구동.
- **독립 데몬 제어:** 시스템 서비스(systemd) 등록 없이 `/usr/sbin/apache2 -d [경로] -f [설정파일] -k start/stop` 명령어를 활용한 직접적인 데몬 프로세스 통제.

## 4. 세부 기능 구현 목록

### 4.1 로그인 및 회원 기능
- **회원가입 및 인증:** 사용자 식별 정보(아이디, 비밀번호 등) 입력 및 저장 (패스워드 단방향 암호화 적용)
- **로그인/로그아웃:** 세션(Session) 객체를 활용한 인증 상태 관리 및 접근 제어
- **마이페이지:** 인증된 사용자 본인의 등록 정보 열람 및 개인정보 수정 기능

### 4.2 게시판 및 콘텐츠 기능
- **게시글 관리(CRUD):** 게시판 목록 페이징 처리, 게시글 작성, 소유자 검증을 통한 수정 및 삭제 기능
- **상호작용 기능:** 게시글 열람 시 조회수 증가, 사용자 간 추천(Like) 프로세스 구현
- **댓글 시스템:** 특정 게시글에 종속된 일반 댓글 작성 및 대댓글(계층형/Tree) 구조 설계
- **파일 관리:** 게시글 작성 시 로컬 스토리지에 첨부파일 업로드 및 열람 시 스트림 다운로드 지원

### 4.3 공지사항 기능
- **공지사항 처리:** 일반 게시판 데이터와 구분된 별도의 상단 고정 리스트 제공
- **권한 분리:** 일반 사용자는 조회 전용, 관리자 계정만 등록 및 수정 가능하도록 권한(Authorization) 로직 분리

### 4.4 관리자 기능
- **계정 관리:** 전체 시스템 가입자 리스트 모니터링, 비정상 사용자 계정 차단 및 강제 삭제
- **콘텐츠 검열:** 시스템 내 등록된 모든 게시글 및 댓글을 관리자 권한으로 직접 강제 수정 및 삭제 처리
- **운영 공지:** 메인 페이지 또는 게시판 상단 노출용 공지사항 전용 글의 배포 및 관리

## 7. 시스템 데이터베이스 설계 가이드 (ERD 구조 개요)
- **Member (회원):** 고유식별번호(PK), 로그인 ID, 암호화 패스워드, 사용자 실명, 권한 등급 플래그
- **Post (게시글):** 게시글 번호(PK), 제목, 텍스트 데이터, 작성자_ID(FK), 작성/수정 Timestamp, 조회 카운트
- **Comment (댓글):** 댓글 번호(PK), 귀속게시글_ID(FK), 작성자_ID(FK), 댓글 내용, 부모 계층형 번호(Self-FK)
- **File (첨부파일):** 파일 고유번호(PK), 귀속게시글_ID(FK), 실제 파일명, 서버 저장 난수경로, 파일 크기 및 속성
