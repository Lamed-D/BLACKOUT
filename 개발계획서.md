# 프로젝트 개발계획서 (Project Development Plan)

## 1. 프로젝트 개요
**1.1 프로젝트 명**
- 미정 (웹 서비스 구축 프로젝트)

**1.2 개발 목적**
- 웹서비스의 전체 구조(프론트엔드-백엔드-DB)에 대한 이해 및 실무 역량 강화
- 회원, 게시판, 관리자 기능을 포함한 핵심 비즈니스 로직 직접 설계 및 구현
- 웹 취약점 분석 및 보안 학습이 가능한 실습용 서비스 구축

**1.3 기대 효과**
- 웹서비스 개발의 전반적인 흐름 이해 및 실제 서비스 수준의 기능 구현 경험 확보
- 향후 웹 보안 및 취약점 분석 학습에 활용할 수 있는 기반 시스템 마련

---

## 2. 개발 환경 및 기술 스택
**2.1 기술 스택 (권장/예시)**
- **Frontend:** 자유 선택 (HTML/HTML5, CSS, JavaScript, 필요시 React/Vue 등 프레임워크 도입)
- **Backend:** 자유 선택 (Python(Flask/Django/FastAPI), Node.js, Spring Boot 등)
- **Database:** RDBMS (MySQL, PostgreSQL, MariaDB 등)
- **Infrastructure:** Docker 컨테이너 환경

**2.2 개발 시스템 요구사항**
- 멀티 플랫폼 웹 브라우저 호환
- 컨테이너 기반(Docker) 배포 파이프라인 구축을 통한 독립적 실행 보장

---

## 3. 사용자 및 권한 구분
**3.1 일반 사용자**
- 서비스의 주요 기능을 이용하는 회원 (가입, 로그인, 게시글 관리, 파일 업로드/다운로드 등)

**3.2 관리자**
- 시스템 전반을 관리하는 최고 관리자 (회원 계정 통제, 콘텐츠 관리, 공지사항 등록 등)

---

## 4. 세부 기능 구현 목록

### 4.1 회원 관련 기능
- **회원가입 및 인증:** 아이디, 비밀번호, 기본 정보 입력 및 저장
- **로그인/로그아웃:** 세션 또는 토큰 기반 인증 제공
- **계정 찾기:** 아이디 찾기 및 비밀번호 초기화/재설정 기능
- **마이페이지:** 회원 자신의 등록 정보 조회 및 수정

### 4.2 게시판 및 콘텐츠 기능
- **게시글 관리:** 게시글 목록 조회, 작성, 수정, 삭제 기능 구현
- **부가 기능:** 게시글 열람 시 조회수 증가, 사용자 간 추천 기능
- **댓글 시스템:** 게시글에 대한 일반 댓글 및 대댓글(계층형) 작성/수정/삭제
- **파일 관리:** 게시글 작성 시 첨부파일 업로드 및 열람 시 다운로드 지원
- **선택적 추가 기능:** 마크다운(Markdown) 문법 지원 또는 텍스트 에디터 플러그인 연동

### 4.3 공지사항 기능
- **공지사항 처리:** 일반 게시글과 구분되는 별도의 공지사항 리스트 제공
- **공지사항 열람:** 일반 사용자는 조회만 가능하도록 권한 분리

### 4.4 관리자 기능
- **회원 관리:** 시스템 가입자 전체 목록 조회 및 정보 수정, 차단/삭제 권한
- **콘텐츠 모니터링:** 시스템 내 모든 게시글 및 댓글을 검열, 수정, 삭제 가능
- **공지사항 관리:** 공지사항 전용 글의 작성, 수정, 삭제
- **계정 권한 관리:** 다른 사용자를 관리자로 승격하거나 권한 해제

## 5. UI/UX 화면 구성 계획
1. **메인 페이지:** 서비스 소개, 로그인 동선 제공
2. **인증 관련 페이지:** 회원가입 폼, 아이디/비밀번호 찾기 폼, 로그인 페이지
3. **게시판 및 콘텐츠:** 게시글 목록, 공지사항 목록, 게시글 상세(댓글 표시), 컨텐츠 작성 및 수정 통합 화면
4. **관리자 대시보드:** 관리자 전용 통계 표출, 회원 및 게시글 일괄 관리 리스트

---

## 6. 데이터베이스 설계 방향 (ERD 구성 가이드)

- **회원(Member):** 회원번호(PK), 아이디, 비밀번호, 이름, 이메일, 권한(일반회원/관리자)
- **게시글(Post):** 게시글번호(PK), 제목, 내용, 작성자(FK:회원번호), 작성일, 수정일, 조회수, 추천수
- **댓글(Comment):** 댓글번호(PK), 게시글번호(FK:게시글번호), 작성자(FK:회원번호), 내용, 작성일, 부모댓글번호(Self-FK)
- **첨부파일(File):** 파일번호(PK), 게시글번호(FK:게시글번호), 파일명, 저장경로, 업로드일
- **공지사항(Notice):** 공지번호(PK), 제목, 내용, 작성자(FK:회원번호), 작성일

---

## 7. 시스템 아키텍처 및 구현 제약사항
- 데이터 무결성 보장을 위해 연관된 데이터(예: 게시글 삭제 시 관련된 댓글/첨부파일 처리)에 대한 외래키(ForeignKey) 정책 수립 필수
- Dockerize를 위해 `Dockerfile`과, 백엔드 및 DB 컨테이너를 함께 띄우는 `docker-compose.yml` 구성 필요
